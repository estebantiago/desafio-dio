Client Side

< voltar

Client Side é tudo o que acontece do lado do usuário, ou seja, no navegador. Quando a gente entra em um site, o que a gente vê e interage – como os botões, imagens, cores e textos – é o que chamamos de Client Side.

É como se fosse a “parte da frente” do site, feita com HTML, CSS e JavaScript. Por exemplo, quando clicamos em um botão e aparece uma mensagem na tela sem recarregar a página, isso é uma ação feita do lado do cliente, sem precisar conversar com o servidor.

Tudo isso acontece no nosso próprio computador ou celular, e não no servidor do site. Por isso, o Client Side ajuda o site a ser mais rápido e interativo, mas ele não deve ser usado para coisas que precisam de segurança, como login ou senhas, porque o código fica visível para qualquer pessoa.

Server Side

< voltar

Server Side é tudo o que acontece no servidor, que é como o “cérebro” por trás do site. Quando a gente acessa uma página, o navegador manda um pedido pro servidor, e é lá que o site processa as informações, busca dados no banco de dados e monta o que vai ser mostrado pra gente.

Essa parte é feita com linguagens como PHP, Python, Node.js, Java, entre outras. A gente não vê esse código, porque ele roda escondido no servidor. Por exemplo, quando fazemos login ou buscamos um produto em uma loja, é o Server Side que verifica se está tudo certo e envia a resposta para o navegador.

É a parte mais segura do site, onde ficam as regras, os dados importantes e tudo que não deve ser exposto. Depois que o servidor faz o trabalho, ele envia um HTML pronto pro navegador mostrar.

HTML

< voltar

HTML é a sigla para HyperText Markup Language, que em português significa “Linguagem de Marcação de Hipertexto”. Ele serve para dizer ao navegador o que é cada coisa: onde vai um título, um parágrafo, uma imagem, um botão, e assim por diante.

É como se o HTML fosse o esqueleto da página. Só com HTML a página funciona, mas fica toda simples e sem cor. Por isso, a gente usa CSS para deixar bonita e JavaScript para fazer ela interagir

Tags

< voltar

As tags são como instruções que usamos no HTML para construir a estrutura de um site. Elas dizem ao navegador o que é cada parte da página, como títulos, parágrafos, imagens, links, entre outros.

Cada tag tem uma função específica. Algumas servem para mostrar texto, outras para inserir imagens, vídeos ou até organizar o layout. Usar as tags corretamente ajuda a deixar a página clara, organizada e fácil de entender, tanto para o navegador quanto para os usuários.

As tags geralmente vêm em pares: uma para abrir e outra para fechar o conteúdo. Mas algumas funcionam sozinhas, sem fechamento. Com as tags, conseguimos montar tudo que aparece visualmente em um site, e também dar significado ao conteúdo.

Atributos

< voltar

Os atributos em HTML são informações extras que colocamos dentro das tags para especificar algum detalhe ou comportamento. Eles ajudam a configurar melhor como cada elemento deve funcionar ou aparecer na página.

Por exemplo, um atributo pode dizer qual é o endereço de uma imagem, o tamanho de um botão, o texto que aparece quando passamos o mouse, ou até se um campo de formulário é obrigatório. Cada tag tem seus próprios atributos possíveis, e alguns são usados com muita frequência em várias partes do HTML.

Os atributos são sempre escritos dentro da tag de abertura e têm dois lados: o nome do atributo e o valor que ele deve ter. Eles deixam o HTML mais completo e flexível, pois permitem ajustar as coisas com mais precisão.

Textos

< voltar

No HTML, os textos são uma das partes mais importantes de qualquer página. Eles formam o conteúdo que as pessoas leem, como títulos, parágrafos, descrições, links e muito mais.

Para organizar e mostrar os textos corretamente, usamos tags específicas, como as de título e as de parágrafo. Cada uma tem sua função e ajuda a deixar o site bem estruturado. Com isso, o navegador sabe o que é um texto importante, o que é só uma explicação, e até o que deve aparecer em negrito ou itálico.

Além da estrutura, o texto também pode ser formatado com estilo, usando CSS. Assim, dá pra mudar a cor, o tamanho da letra, o tipo de fonte e o espaçamento, deixando a leitura mais agradável.

Listas

< voltar

As listas em HTML servem para organizar informações de forma clara e fácil de ler. Elas são muito úteis quando queremos mostrar itens em sequência, como passos, tópicos, opções ou qualquer coisa parecida com uma lista de supermercado ou um menu.

Existem dois tipos principais de listas: uma que usa números para mostrar a ordem dos itens, e outra que usa bolinhas ou marcadores quando a ordem não importa. Também existe um tipo de lista onde podemos relacionar pares de termos e definições, como em um glossário

Links

< voltar

Os links em HTML são usados para conectar uma página a outra. Eles permitem que a gente navegue entre diferentes partes de um site ou até vá para outros sites. Também podem ser usados para abrir um e-mail, fazer download de um arquivo ou abrir algo em outra aba do navegador.

Para criar um link, a gente usa uma tag específica que indica o endereço para onde o link vai levar. Esse endereço pode ser de uma página do mesmo site ou de um site diferente.

Além disso, dá para configurar o link para abrir na mesma aba ou em uma nova aba do navegador. Isso é útil quando a gente quer que o usuário continue com a página atual aberta enquanto acessa outro conteúdo.
